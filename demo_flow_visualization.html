🔐 Voice of Customer Platform - User Authentication Flow

1
User Visits Homepage
👤 User Action: Navigate to platform URL
URL: http://your-platform.replit.app

What User Sees:

  • Homepage with Arabic/English bilingual interface
  • Navigation dropdown: "تسجيل الدخول" (Login in Arabic)
  • Main CTA button: Login option prominently displayed
  • Demo features accessible without login

User State: Anonymous/Guest visitor

2
Authentication Initiation
👤 User Action: Clicks login button

System Response:

  1. Redirects to /auth/replit_auth
  2. Flask-Dance initiates OAuth 2.0 flow
  3. Generates secure PKCE challenge
  4. Redirects to Replit OAuth endpoint
OAuth Redirect URL: https://replit.com/oidc/auth? response_type=code& client_id=b180e878-e611-47a1-9094-48a39eeb9914& redirect_uri=http://platform.replit.app/auth/replit_auth/authorized& scope=openid+profile+email+offline_access& state=& code_challenge=& code_challenge_method=S256& prompt=login+consent
OAuth Parameters:
• client_id: Unique app identifier registered with Replit
• scope: openid (identity) + profile (name/picture) + email + offline_access (refresh token)
• PKCE: Proof Key for Code Exchange for enhanced security
• state: Random token to prevent CSRF attacks
3
Replit Authentication
👤 User Action: Authenticate with Replit

User Experience on Replit:

  • If no Replit account: User creates new account on replit.com
  • If existing user: Signs in with Replit credentials
  • Authorization prompt: User grants permission for app to access profile data

Permissions Requested:

  • 🆔 openid: Basic identity verification
  • 👤 profile: Name and profile picture
  • 📧 email: Email address
  • 🔄 offline_access: Refresh token for persistent login
4
OAuth Callback & User Creation
🔄 System Action: Process OAuth callback

System Operations:

1. Create User Record (replit_users)
• id: "user_12345" (Replit user ID)
• email: "john.doe@example.com"
• first_name: "John"
• last_name: "Doe"
• profile_image_url: "https://replit.com/.../profile.jpg"
• created_at: "2025-01-15 10:30:00"
2. Create Preferences (replit_user_preferences)
• user_id: "user_12345"
• language_preference: "ar" (Arabic default)
• timezone: "Asia/Riyadh"
• theme: "light"
• is_admin: false
• admin_level: "user"
3. Store OAuth Tokens (replit_oauth)
• user_id: "user_12345"
• provider: "replit_auth"
• token: {access_token, refresh_token, expires_at}
• browser_session_key: "session_abc123"
5
Authenticated User Experience
✅ User Successfully Authenticated

User Now Has Access To:

  • 🏠 Personalized Homepage: Shows user's name and profile picture
  • 👤 Profile Management: View/edit platform preferences at /profile
  • 📊 Analytics Dashboard: Full access to insights and reports
  • 📝 Survey Management: Create, edit, and manage surveys
  • 🔧 Platform Features: All authenticated user features

Navigation Changes:

  • Login button → User profile dropdown
  • Shows user's name and profile picture
  • Logout option available
  • Access to settings and preferences
6
Admin Elevation (If Needed)
👑 Admin Action: Promote user if needed

Admin Promotion Process:

  1. Existing admin accesses /settings/users
  2. Finds the new user in the management interface
  3. Uses "Make Admin" action to promote user
  4. System updates user preferences: is_admin = true

Available Admin Levels:

  • 🔵 user: Standard platform access
  • 📊 analyst: Advanced analytics features
  • ⚙️ admin: User management, system settings
  • 👑 super_admin: Full system control
